# 품질진단 시스템 — 리디자인 (Streamlit)

기존 `quality-diagnosis` 레포의 디자인을 **공공/행정 톤(딥 네이비 + 톤다운된 인스티튜셔널 블루)**으로
재디자인한 버전입니다. **기능 로직·세션 상태·파일명·st.navigation 구조는 원본과 100% 동일**하며,
오직 스타일(색·폰트·헤더·차트 팔레트)만 교체했습니다.

## 적용 방법 (드롭인 교체)

이 폴더의 파일을 원본 레포의 같은 경로에 덮어쓰기 하세요.

```
quality-diagnosis/
├── app.py                      ← 교체 (전역 네이비 테마 + 사이드바)
└── pages/
    ├── 0_home.py               ← 교체
    ├── 1_project.py            ← 교체
    ├── 2_data_connect.py       ← 교체
    ├── 3_diagnosis_set.py      ← 교체
    ├── 4_diagnosis_run.py      ← 교체
    ├── 5_result.py             ← 교체
    ├── 6_report.py             ← 교체
    ├── 7_erd.py                ← 교체
    └── 8_standard.py           ← 교체
```

`core/`, `templates/`, `wallet/`, `requirements.txt` 등 나머지는 그대로 두면 됩니다.

```bash
streamlit run app.py
```

## 디자인 시스템

| 토큰 | 값 | 용도 |
|---|---|---|
| Navy | `#102a4c` / `#0e2340` | 사이드바, 히어로, 강조 |
| Blue | `#2e5aa8` | 포인트 컬러, 차트, 탭 인디케이터 |
| BG | `#eef1f6` | 본문 배경 |
| Border | `#dfe4ec` | 카드/테이블 보더 (radius 5px) |
| Font | Pretendard | 전 화면 본문 |

- **6대 품질영역 카테고리 컬러** (채도 낮춘 6색):
  완전성 `#1f4e87` · 일관성 `#2c7a6b` · 정확성 `#c0851e` · 유용성 `#5a6b8c` · 유일성 `#7a4a78` · 유효성 `#b04a3e`
- 레드(`#b23b3b`)는 오류/위험 등급 배지에만 최소 사용
- Plotly 차트는 투명 배경 + Pretendard + 인스티튜셔널 팔레트로 통일

## 변경 요약

- `app.py`: 전역 CSS 디자인 시스템 주입(폰트/배경/카드/버튼/탭/메트릭/테이블/입력), 사이드바를 네이비 그라데이션 + 프로젝트 카드 + 진행 게이지 바로 리디자인
- `0_home.py`: 그라데이션 헤더 → 네이비 히어로, 막대그래프·6대 영역 카드 리디자인
- `3·4·5·7·8`: 각 페이지 자체 CSS의 구(舊) 팔레트(보라/형광)를 네이비 팔레트로 교체, 차트·배지·하이라이트 색 통일
- `1·2·6`: 헤더 통일(네이비 eyebrow 라벨), 차트 팔레트 정리

> 기능 코드는 한 줄도 바꾸지 않았습니다. (DB 연결, 진단 엔진, 보고서 생성, AI 호출 모두 원본 그대로)
